Design Rationale: Personal Portfolio

Users

The primary intended users of this website would be any potential future employers, but the user population could extend to any colleague or classmate that wants to learn more about me and my interests. This website would then allow them to learn relevant information about me, my experience and my skills, focusing on my study in Computer Science and its related fields.

Usability Dimensions

I prioritized efficiency in the design of the website, while also ensuring that other usability dimensions were satisfied.

  • Learnability & Memorability: the website has its pages and the links that access them spelled out in full, so users know which page they are navigating to when they click the various links on the website.
  • Efficiency: all "main" pages are accessible from any other page through the navigation bar at the top of the website; users can navigate to the relevant pages for its listed information whenever they wish to. Some "secondary" pages (this rationale page, for instance) also have multiple access links: this page can be reached from both the Home page and the Projects page. Information on the pages is also presented as simply as possible, so the user can easily understand the description on e.g. one of the project cards.
  • Satisfaction: small satisfaction elements such as button and link hover colors are consistent across the website, and relevant icons are present wherever appropriate.

Design Specifications

Color Scheme

I chose a dark blue for the primary color; this choice is in part due to personal preference (darker themes are easier on the eyes for me), but I also feel that while a dark blue gives off a proper and professional feeling, the slight green tint of the color prevents it from being overly "cold."

The secondary color is an even darker blue that makes use of Bootstrap's "dark" color scheme. It is of a similar hue to the primary color of the website, so it does not seem out of place.

The accent color is a pale yellow (mostly showing up as a gradient with white), based on it being a complementary color of the primary color chosen. The gradient creates greater visual interest in the webpages.

Text color is mostly white to contrast with the dark background colors, and gray denotes less-important text in the page.

Layout

The layout is kept similar to that of the template: mostly, information is presented in the form of cards on the page. This allows information about any particular topic (one past work experience, one project etc.) to appear contained, so it is easy to understand that this information all points to a single topic for each card.

Images

The Projects page contains images that are associated with the main topic of each project, so it is easy at a glance to understand what kind of work I have done and am interested in doing.

In the case study for the CS 220 final project, I include images like the Figma canvas that showcase the process of creating our prototypes, in addition to the video demo showcasing our app's full functionality. This provides an easy visual representation of the app, so that users can understand more about the project.

The image of myself on the Home page was chosen rather arbitrarily; I figured that any picture of me that felt reasonably appropriate to show in a professional setting would be alright.

Template Modifications & Resources

Most changes made to the template were out of personal preference. The changes to colors are for the reasons stated above, and I altered the positioning of elements in the pages slightly, mostly concentrating elements closer to the center of the screen so that the user does not have to look across their entire screen to read the contents of the page (this is most apparent in a back-to-back comparison of the Home pages).

As shown above, the "About Me" section is cut off (due to the greater height of the image used), so I added a small downward-pointing icon to indicate that the user can scroll further in the page.

I also altered the cards to have more concentrated content for a similar reason. The Skills section of the Resume page also no longer contains internal "pills" that house each individual skill, since the color scheme change made them seem like clickable elements due to the high contrast. I also added some information I felt could be relevant to potential users of the website, such as dates of each project in the Projects page. The Education section now includes a link to Wellesley's website, rather than a text description, since I feel the school can better speak for itself (though this also results in an extra click for the user - I've attempted to mitigate this consequence by having the website open in a new browser tab).

On resources used, I consulted w3schools for some unfamiliar HTML uses like embedding videos and having links open in a new tab, etc. Images are either screenshots from our project or sourced royalty-free from Pixabay. Naturally, Bootstrap constitutes a significant resource used in this website as well. One additional resource I should note is this StackOverflow discussion - there was a small area of white space on the right side of the page that bothered me slightly as it allowed the page to scroll sideways, so I used the suggested solution to fix it.

GenAI in Web Development

While I prefer not to use it if I am able to find a solution on my own, I think generative AI could potentially be helpful for ideation in layouts and color schemes, as well as generating a basic template to reference while implementing the website.

However, as we realized in class, generative AI HTML code will sometimes contain errors that can be varyingly difficult to identify and fix by hand. Using such tools to generate the website also forms a barrier to understanding more complex parts of the code, which makes maintenance of the website difficult - if we do not understand the AI code output, we will have trouble finding the appropriate part of the code to update.
